Arduino: Sensor de distancia con alerta

Hardware • C++ • Principiante

En este proyecto construí un sistema simple de detección de proximidad. Utilizando un sensor ultrasónico HC-SR04, medimos la distancia a un objeto y usamos un buzzer para emitir una alerta sonora que se intensifica cuanto más cerca está el objeto, similar al sensor de reversa de un automóvil.

Materiales necesarios

  • Placa Arduino Uno (o compatible)
  • Sensor ultrasónico HC-SR04
  • Led
  • Protoboard y cables jumper
  • Resistencia de 220Ω (opcional, para proteger el led)

El Código

La lógica es sencilla: enviamos un pulso ultrasónico, medimos cuánto tarda en volver (eco) y calculamos la distancia. Si la distancia es menor a 30cm, activamos el buzzer. 


// Definición de pines
const int trigPin = 9;
const int echoPin = 10;
const int buzzerPin = 11;

void setup() {
  pinMode(trigPin, OUTPUT); // El trigger envía el pulso
  pinMode(echoPin, INPUT);  // El echo recibe el rebote
  pinMode(buzzerPin, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  // 1. Limpiar el pin del trigger
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  
  // 2. Enviar pulso de 10 microsegundos
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);
  
  // 3. Leer el tiempo del eco
  long duration = pulseIn(echoPin, HIGH);
  
  // 4. Calcular distancia (velocidad sonido = 0.034 cm/us)
  int distance = duration * 0.034 / 2;
  
  // 5. Lógica de alerta
  if (distance < 30 && distance > 0) {
    // Mapear distancia a frecuencia: más cerca = tono más agudo
    int toneFreq = map(distance, 0, 30, 1000, 200);
    tone(buzzerPin, toneFreq);
    
    // Opcional: imprimir en monitor serial
    Serial.print("Alerta! Distancia: ");
    Serial.println(distance);
  } else {
    noTone(buzzerPin);
  }
  
  delay(100); // Pequeña pausa para estabilidad
}

Explicación técnica

El sensor HC-SR04 funciona como un sonar. Emite un sonido inaudible para el oído humano (ultrasonido). La función pulseIn() de Arduino mide el tiempo en microsegundos que el pin echoPin permanece en estado ALTO, lo que corresponde al tiempo de vuelo de la onda sonora.

Dividimos por 2 porque el sonido viaja de ida y vuelta. La constante 0.034 es la velocidad del sonido en cm/µs.
